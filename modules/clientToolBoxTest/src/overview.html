The clientToolBox test

What is tested?

This is a test of clientToolBox. clientToolBox is a module of EJBCA not running inside the application server. It is a stand alone CLI application.

clientToolBox can be used for the following things:

      Doing some tests and object manipulations of a p11 device (could be a HSM).
      Testing some of the protocols that EJBCA are supporting.

How is the test done?

Description on how to run tests is found in README. Read it if you have not done it yet. The directory of the README will be referred to as $TEST_HOME in this documentation. This is also the './dist' directory created when building the source (ant).

The TestNG framework is used to run the tests. Each test suite is defined with a TestNG xml file.

The xml files for all predefined suites are located in the $TEST_HOME/resources/suites directory.

Other suites than the predefined could be used by using the name of an xml file instead of the predefined suite name when starting the test.

All test classes referred to by suites belongs to the package {@link org.ejbca.ui.cli.clientToolBoxTest.tests}. A test in a test suite is defined by specifying one or several of these test classes with parameters.

The start of the testing is done by classes in the package {@link org.ejbca.ui.cli.clientToolBoxTest.start}.
